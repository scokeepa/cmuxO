# JARVIS Red Flags

| 이런 생각이 들면 | 실제로 해야 하는 것 |
|-----------------|-------------------|
| "이건 간단해서 테스트 안 해도 돼" | TDD 필수 (Iron Law #2) |
| "내가 만든 거니까 잘 됐을 거야" | 독립 검증 스크립트 실행 (Iron Law #3) |
| "사용자한테 물어보면 귀찮아할 거야" | 반드시 승인 요청 (Iron Law #1) |
| "이전에 비슷한 게 잘 됐으니까" | 새로 검증 (Iron Law #3) |
| "롤백하면 되니까 바로 적용하자" | 백업 먼저, 적용은 승인 후 |
| "빨리 해야 하니까 GATE 우회하자" | GATE는 hook으로 강제됨, 우회 불가 |
| "free-text를 승인으로 해석하자" | 구조화 선택지 재요청 |
| "새 최상위 키를 추가하자" | 기존 키 내부에서만 수정 |
